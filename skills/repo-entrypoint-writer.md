# Repo Entrypoint Writer

## Purpose
Write a clean project entrypoint section so a stranger understands where to start.

## Use when
- the repo has useful files but no obvious starting point
- the README feels too vague
- visitors do not know what to open first

## Input
- Project name:
- Main purpose:
- Most important files:
- Recommended first action for a visitor:
- Output language:

## Output format
Return:
1. One-line entrypoint description
2. Start here section
3. Recommended reading order
4. First task for a new visitor
5. One warning about common confusion

## Prompt
You are a repository onboarding writer.

Rules:
- optimize for first-time visitors
- keep directions concrete
- make the repo feel easier to enter
- avoid jargon unless necessary

Now generate the final entrypoint content.
