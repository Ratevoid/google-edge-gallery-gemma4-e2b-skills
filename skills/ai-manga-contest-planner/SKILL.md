# AI Manga Contest Planner

## Purpose
Use this skill when the user wants help with:
- AI漫剧比赛
- AI comic or animated comic contest planning
- choosing a safer topic
- increasing award probability
- balancing speed versus quality
- deciding story scope
- making a shot list or scene list
- turning vague ideas into a production workflow

## Priority logic
Default priorities should be:
1. finishability
2. readability
3. theme fit
4. judging impact
5. technical ambition

Do not optimize for complexity first.
A complete, emotionally clear, coherent short work is better than an over-scoped mess.

## Output format
Always structure the answer with these sections:
1. Contest objective
2. Core concept
3. Why this concept is competitive
4. Story structure by scene
5. Visual production strategy
6. AI tool chain suggestion
7. Time-boxed execution plan
8. Risk points and backup plan
9. Submission checklist
10. Next action in the next 2 hours

## If information is missing
If the user does not provide contest rules, explicitly say what is assumed.
If the user does not provide a deadline, assume a short-form production sprint and optimize for a compact piece.
If the user has multiple goals, separate them into:
- highest award probability
- fastest completion
- strongest portfolio value

## Rules
- Push toward fewer scenes, clearer emotion, stronger hook.
- Avoid stories that require too many unique environments or character redesigns.
- Prefer repeatable visual language and modular scene assembly.
- For weak schedules, recommend 1 protagonist, 1 strong conflict, 1 visual motif, 1 payoff.
- When relevant, suggest how to hide AI inconsistency through framing, lighting, texture, and edit rhythm.
